# OpenCart Registration Module Testing

## Project Overview

This project demonstrates comprehensive manual testing of the Registration Module of the OpenCart E-commerce Application. The objective was to verify the functionality, validation rules, usability, and reliability of the user registration process.

## Application Under Test (AUT)

OpenCart E-commerce Application

## Module Tested

* User Registration Form
* Input Field Validations
* Mandatory Field Checks
* Password Validation
* Email Validation
* Privacy Policy Agreement
* Successful Account Creation

## Testing Activities Performed

* Requirement Analysis
* Test Planning
* Test Case Design and Execution
* Requirement Traceability Matrix (RTM) Creation
* Defect Identification and Reporting
* Test Documentation

## Deliverables

### Test Plan

Prepared a detailed test plan including scope, objectives, testing approach, resources, and entry/exit criteria.

### Test Cases

Designed positive, negative, and validation test scenarios for the registration functionality.

### Requirement Traceability Matrix (RTM)

Mapped requirements with corresponding test cases to ensure complete test coverage.

### Bug Report

Documented identified defects with severity, priority, expected results, actual results, and reproduction steps.

## Test Scenarios Covered

* Registration with valid data
* Registration with empty mandatory fields
* Registration with invalid email format
* Registration with duplicate email address
* Password and Confirm Password mismatch
* Password validation checks
* Privacy Policy not selected
* Successful account creation
